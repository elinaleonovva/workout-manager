# Инструкция по применению миграций

## Для добавления новой модели WorkoutPlan

После добавления модели `WorkoutPlan` в `backend/api/models.py` необходимо создать и применить миграции:

### Шаги:

1. **Создать миграцию:**
   ```bash
   cd backend
   python manage.py makemigrations
   ```

2. **Применить миграцию:**
   ```bash
   python manage.py migrate
   ```

### Если используете Docker:

```bash
docker-compose exec backend python manage.py makemigrations
docker-compose exec backend python manage.py migrate
```

Или пересоберите контейнеры:
```bash
docker-compose down
docker-compose up -d --build
```

Миграции применятся автоматически при запуске контейнера (если настроено в docker-compose).

## Что было добавлено:

- **Модель WorkoutPlan** - для хранения планов тренировок с периодичностью
- **API endpoint** `/api/workout-plans/` - для работы с планами
- **Компонент Calendar** - календарь тренировок на фронтенде
- **Обновлен MyWorkouts** - возможность создания планов тренировок

## Функционал:

1. Пользователь выбирает упражнение из списка (которые добавляет администратор)
2. Указывает дату первой тренировки
3. Указывает периодичность (дней между тренировками)
4. Система автоматически рассчитывает следующие даты тренировок
5. Все даты отображаются в календаре
